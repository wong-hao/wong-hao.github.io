Rejeee SX1302S and M-HL9-EV

Oct 6, 2020
Slides
Photo by rawpixel on Unsplash

1.实验环境

1.1 操作系统

Raspberry Pi OS (32-bit) with desktop and recommended software

注意仅能使用Linux Kernel version: 4.19.97及以下的版本，否则gpio7将会占用从而一直不能正确启动

// 报错提示
./reset_lgw.sh: 22: echo: echo: I/O error
./reset_lgw.sh: 25: ./reset_lgw.sh: cannot create /sys/class/gpio/gpio7/direction: Directory noexistent
CoreCell reset through GPIO7...
./reset_lgw.sh: 32: ./reset_lgw.sh: cannot create /sys/class/gpio/gpio7/value: Directory noexistent
./reset_lgw.sh: 33: ./reset_lgw.sh: cannot create /sys/class/gpio/gpio7/value: Directory noexistent
ERROR: failed to connect
// 或者是
ERROR: [main] failed to start the concentrator

Gitee版本代码相比 Github做了三点改变： 其实Gitee之前的commit全部是copy的Github，就最后改了下

  1. 使用#if USE_I2C_SENSOR ... #endif消除温度传感器I2C（据说不注释掉不能启动）
  2. boardconf.lorawan_public = true;改成了boardconf.lorawan_public = false; （完全不影响）
  3. SX1302_RESET_PIN=23改成了SX1302_RESET_PIN=7且去掉了GPIO18相关代码 （GPIO7存在才能启动，GPIO18改不改没关系）

1.2 备份

  1. Gitee版采用的代码Commit id（选用自动安装脚本’install.sh’）: d9ef27fdecf2352d7ab6b85d50fb7a3e38a162ae
  2. Github版采用的代码Commit id（Gitee抄到这后面就自己改了）: 6291e62ef9a161ae41037a0df1a86e9943a907b9
  3. 历代系统
  4. 4.19.97系统镜像

1.3 M-HL9-EV

  • 按键
    • 模式切换
      • 透传模式要先使用+++进入AT模式
      • AT模式可直接使用AT指令
      • （AT指令需且仅需在指令后空一行再发送，但与’自动换行’无关，那个是屏幕上显示的东西换行）
      • AT模式下不能发送接收数据，需要使用AT指令ATT返回透传模式
    • 复位按键
      • 重置串口

2.实验步骤

2.1硬件

  1. sx1302连直黑天线
  2. 官方建议 M-HL9-EV连金黄天线，但实测用直黑天线进行接受测试效果更好
  • 使用树莓派搭建网关注册到腾讯云
    • 主要步骤
      • 官网
    • 补充
      • 获取GwEUI，即在gw1302s/bin下使用./chip_id，注意还要用该值替换掉gw1302s/bin/global_conf.json中的gateway_ID，且需要去掉开头的0x，这是代表十六进制的开头
      • 运行./lora_pkt_fwd即可看到 状态在线
      • 云处理数据
    • 作用
      • 判断是否真正上网了，或者说真正处理数据了
    • 注意
      • Gitee版本直接增加了一个文件gw1302s/packet_forwarder/global_conf.json，并将其中的"gateway_conf"下面的server_address 默认值localhost修改为了腾讯云
      • 卖家给的 攻略有另一种global_conf.json的思路

2.2源码

  1. gw1302s/libloragw
  2. gw1302s/packet_forwarder
  3. gw1302s/tools/payload_tools
  4. gw1302s/util_net_downlink
  5. gw1302s/util_chip_id

五个Helper program在主目录下，通过每个Helper下的makefile文件将必要的文件scp到../target.cfg中的TARGET_DIR中

故gw1302s/bin下看不懂的文件去看对应Helper program下的Readme.md，但执行还是在这里执行
