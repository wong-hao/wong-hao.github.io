1 实验环境

1.1 操作系统

Raspberry Pi OS (32-bit) with desktop and recommended software

注意仅能使用Linux Kernel version:4.19.118及以下的版本，否则由于硬件使用GPIO7作为SX1302的RESET，与树莓派的SPI0.1的CS脚冲突

Gitee版本代码相比Github做了三点改变： 其实Gitee之前的commit全部是copy的Github，就最后改了下

  1. 使用#if USE_I2C_SENSOR ... #endif消除温度传感器I2C（据说不注释掉不能启动）
  2. boardconf.lorawan_public = true改成了boardconf.lorawan_public = false （完全不影响）
  3. SX1302_RESET_PIN=23改成了SX1302_RESET_PIN=7且去掉了GPIO18相关代码 （GPIO7存在才能启动，GPIO18改不改没关系）
  • 网关射频模组

global_conf.json

  • Gitee版本直接增加了一个文件gw1302s/packet_forwarder/global_conf.json，其特点是gateway_conf下面的server_address由localhost修改为了loragw.things.qcloud.com
  • make install_conf即安装global_conf.json
  • 卖家给的攻略跟用install.sh生成的主要差别在radio_1与radio_2的ferq不同，攻略的效果好；官方的参考文献和攻略还有视频均给出了global_conf.json的参数意义
  • “server_address"与Gateway Bridge运行地址一致，Gateway Bridge可安装在网关上或云端
  • 查看网关是否连接上了网络服务器，tcpdump指令需要通过sudo apt update && sudo apt install tcpdump安装
